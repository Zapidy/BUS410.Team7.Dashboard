# Round 7 — Design Context (v2, post-feedback rebuild)

> The first design (parchment + serif + Brookings register) was rejected by the user as "static, off-putting palette, generic." This second pass commits to a different register entirely: a **tool**, not a brochure. Visual confidence over editorial restraint.

## Users

Primary: BUS 410 professor, classmates — a politically-literate but non-technical audience. Classroom projector at 10ft, then laptop close-read at 18in. The viewer is **smart but not a quant** — they need the visualization to make the finding obvious, not to defer to footnotes.

Secondary: future reviewers / portfolio audience.

Context: this is the demo moment for the project. The professor will judge methodological substance AND whether the tool feels real. Both matter.

Job to be done: the user **explores credit-desert risk** across U.S. tracts, **toggles between Model 1 and Model 2**, and **manipulates policy levers** to see how the model responds. The map is the workspace; everything else supports the map. A separate methodology section explains the work but does not interrupt the tool.

## Brand Personality

Three words: **vivid, opinionated, plainspoken.**

Not academic. Not corporate. Not "civic." This is a serious analytical tool that **looks like someone thought about it for more than fifteen minutes** — closer to The Pudding, FiveThirtyEight's interactive pieces, or Mapbox's showcase galleries than to a Brookings report.

Plainspoken means: labels are direct ("Diagnostic risk", "Influenceable risk"), not jargon-laden. A poly-sci undergrad should understand every label without a glossary.

Opinionated means: the visualization makes a strong claim. The two models are visually disambiguated to the point that the user instantly understands "these are different lenses." Subtle is not a virtue here.

Vivid means: high color contrast, generous scale, real animation, not pastel timidity.

## Aesthetic Direction

- **Theme**: dark. Deep ink-navy background with a slight green-blue chroma so it doesn't feel like cheap pure-black. Dark because (a) the audience answer was "let me decide based on persona," and a tool used in a dim classroom + projector context wants dark, (b) the previous light parchment was rejected.
- **References**:
  - **The Pudding** essays — bold typography, custom palette, the data viz IS the page
  - **Mapbox showcase galleries** — full-bleed maps, opinionated typography, color used as primary communication
  - **FiveThirtyEight interactive pieces** — dense data, accessible labels, custom color
  - **Bloomberg Cities Dashboard** (the good ones) — civic data presented confidently
- **Anti-references** (explicit "do NOT look like this"):
  - **Brookings policy PDFs** (the previous design — rejected)
  - **Pew / Urban Institute civic-tech-bad** (small monospace labels, dual-axis sterility)
  - **Generic SaaS dashboards** (Tableau defaults, bootstrap admin templates)
  - **Bloomberg Terminal black-on-amber** (that's Round 5 — different sibling)
  - **Light parchment + serif body** (banned for this redesign — too genteel for a tool)

### Color system

OKLCH-tuned, dark theme, one strong accent per model so you instantly see which model you're on.

```
Background        oklch(0.15 0.020 240)   deep ink-navy
Surface-1         oklch(0.20 0.020 240)   slightly raised panel
Surface-2         oklch(0.25 0.022 240)   hover/active state
Rule              oklch(0.32 0.022 240)   hairline borders
Text primary      oklch(0.96 0.012 80)    warm off-white (NOT pure white)
Text secondary    oklch(0.70 0.020 240)   dusted blue-gray
Text muted        oklch(0.55 0.020 240)   secondary metadata

Accent — Model 1 (Diagnostic)       oklch(0.80 0.165 85)    saturated MUSTARD/ochre
Accent — Model 2 (Influenceable)    oklch(0.85 0.190 130)   saturated CHARTREUSE
Accent — UI state / focus           oklch(0.68 0.20 25)     saturated CORAL

Choropleth ramp M1   bg-ink → mustard, sequential single-hue
Choropleth ramp M2   bg-ink → chartreuse, sequential single-hue
```

Mustard and chartreuse against deep ink read instantly as "two different things" — that's the point. They are NOT AI-reflex colors (no cyan, no purple, no neon-on-black). They feel printed, considered, slightly weird.

Coral as the third accent is reserved for **active state, focus rings, and "live now" cues** (e.g., the slider thumb when being dragged). Used at most 2-3 places visible at once.

### Typography

Reflex fonts to reject (used last time or in the standard banned list): Source Serif 4, Source Sans, Public Sans, Inconsolata, Inter, DM Sans, Jakarta, Outfit, Plex, Space, Fraunces, Newsreader, Cormorant.

Selected, after browsing fresh:

- **Bricolage Grotesque** (Google Fonts, OFL, free) — display + headlines + big numbers. Variable axis lets us go from punchy headlines (700 weight, tightened tracking) to numeric readouts. Has character — slight slab influence in the terminals — without being designy. Pairs with sans body cleanly.
- **Manrope** (Google Fonts, OFL, free) — body, labels, UI. A workhorse geometric sans with subtle humanist warmth. Reads cleanly at small sizes on dark; not in the reflex list. NOT Inter, NOT DM Sans.
- **Recursive Mono Casual** (Google Fonts, OFL, free) — tabular data + numeric readouts + slider readouts. The "Casual" axis gives it a hand-drawn-feeling, slightly informal terminal aesthetic — NOT IBM Plex Mono, NOT JetBrains Mono. Distinctive.

Type rules:
- Modular scale, fixed `rem` for app UI (no fluid clamp in dashboard chrome)
- Display headings: Bricolage Grotesque 700, tight tracking (-0.015em)
- Big numbers (counters, percentages, AUC): Bricolage Grotesque 600, tabular-nums, much larger than surrounding type
- Body: Manrope 400, line-height 1.55 (dark mode +0.05)
- Tabular data: Recursive Mono Casual 400, tabular-nums always
- ALL-CAPS labels reserved for short kicker text (pill labels, section eyebrows). Tracking +0.05em, smaller than body.

### Layout

- **Map is the page.** The choropleth is the centerpiece, occupying ~75% of vertical viewport at desktop. Other elements support the map.
- Sticky controls (model toggle, scenario sliders) along the right-hand side rail of the map — modular sidebar that travels with you. NOT below the map.
- Methodology section is BELOW the map, accessible via deliberate scroll. NOT inline with the tool.
- Asymmetric layouts. Strong left alignment. NO centered-everything.
- Hairline rules (`oklch(0.32 0.022 240)` 1px) for division. NO cards-with-shadows. NO gradient borders. NO glassmorphism.
- Spacing scale: 4, 8, 12, 16, 24, 32, 48, 64, 96. Use generously around section breaks.

## Motion Direction

The previous design's motion was "editorial cinematic" — that wasn't wrong but it lived inside a static-feeling shell. This redesign treats the WHOLE INTERFACE as alive:

- **Map paints alive** on initial load: tracts fade in choropleth-color from outside-in over 800ms (centroid-distance-staggered), gives the dataset weight when it first renders
- **Toggle Model 1 ↔ Model 2**: choropleth color interpolates smoothly over 500ms `cubic-bezier(0.25, 1, 0.5, 1)` (ease-out-expo). The right-rail sidebar swaps headline numbers with a 250ms cross-fade.
- **Slider drag**: choropleth recolors continuously with rAF interpolation. NOT debounced step changes — actual tracking.
- **Counter tweens**: when a counter changes, count up/down with rAF over 350ms ease-out-quart.
- **Hover on tract**: tooltip appears with NO entry animation (instant) but the tract underlay gains a 1px coral outline.
- **Reduced-motion**: instant updates, no spatial movement.

## Design Principles

1. **The map is the page.** Map dominates visually. Other elements are subordinate. Not a "report with a map in it."
2. **Color does the talking.** Mustard for diagnostic, chartreuse for influenceable. The viewer's eye learns the two models from color before reading any label.
3. **Plainspoken labels.** "Diagnostic risk" not "Probability of becoming a service desert at horizon h+1." A poly-sci undergrad reads every label without a glossary.
4. **Tool-first, methodology second.** The interactive surface is the centerpiece. Methodology lives below, accessible but not interrupting.
5. **Confident, not cute.** Bold typography, saturated palette, strong asymmetric layouts. No emoji, no cute animations, no decorative icons.
6. **Honest interactivity.** When a slider "doesn't move the model much" (because of the ablation finding), the UI is transparent about that — small label showing the actual ΔAP. We do not pretend.

## What this is NOT

- Not a magazine spread. Not a published artifact. Not a brochure.
- Not a Brookings PDF. Not parchment. Not a serif body face.
- Not a generic SaaS dashboard. Not Tableau defaults. Not bootstrap-admin chrome.
- Not Bloomberg Terminal density. (That's Round 5 — they're siblings in different registers.)
- Not "civic-tech bad-version" — small monospace labels, dual-axis sterility on white, tiny dual-axis charts.

A redesigned dashboard at `round7/web/` that:
- Replaces every visual decision from v1
- Map dominates the viewport
- Dark ink background, mustard + chartreuse choropleth ramps
- Bricolage Grotesque + Manrope + Recursive Mono Casual
