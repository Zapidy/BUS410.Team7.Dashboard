    METHODOLOGY · GLOSSARY

    THE FORECASTING PROBLEM

    The model predicts whether a US census tract that is not currently a credit desert will become one within the next year (the H1 transition target). A "credit desert" here means a tract whose CRA-reporter lender count falls in the bottom decile of its (year × rural-or-urban peer group) distribution. We deliberately exclude already-deserts from the supervised set — predicting "will stay a desert" is trivially correct ~95% of the time, so it inflates AUC without doing real forecasting work.

    Positive rate in the labeled set: 2.75% · Test predictions: 1.21M across 8 folds.

    WALK-FORWARD VALIDATION

    Eight chronological folds. Each trains on years 2009→T, validates on T+1, tests on T+2 to T+3 (T ranges 2014→2021). This mirrors deployment: fit on everything we know, predict next year. No future leaks into training.

    LEAKAGE PATHS CLOSED

    • Target autocorrelation — switched from "is desert" (sticky, AUC 0.93) to "becomes desert" (genuine forecast, AUC 0.83).
    • 11 CRA circular features dropped — n_cra_lenders directly defines the target; entries/exits/churn at year T leak too.
    • 14 FDIC proxy features dropped — banks are CRA reporters, so fdic_bank_count cross-correlates ~0.85 with n_cra_lenders. Kept HHI / top-share (concentration shape, not level).
    • ACS publication-lag tightened — vintage ≤ year - 2 (was year - 1). ACS 5-year vintage V is published autumn of V+1.
    • PR / VI excluded from training (small samples, different rural-urban geography).
    • Test windows standardized to 2 years across all folds (was 3 for F1-F7, 2 for F8).

    39 features survive after the 25 circular drops. Honest mean AUC: 0.857.

    CALIBRATION

    Per fold, we fit an isotonic regressor on val-set predictions and apply it to test. On Round 5 it was effectively a no-op — Brier moved 0.0203 (raw) → 0.0201 (calibrated). The model is already well-calibrated post-FDIC-drop. The infrastructure exists for future runs where it might matter more.

    WHY POOLED AUC ≠ HEADLINE AUC

    The two numbers measure different things and are not expected to match — even with no filters applied.

    • Masthead AUC 0.857 ± 0.044 — arithmetic mean of 8 walk-forward fold AUCs (F1=0.882, F2=0.884, F3=0.904, F4=0.913, F5=0.826, F6=0.823, F7=0.794, F8=0.826). Each fold's AUC is computed on its own test set; we average those 8 numbers to get the headline. This is the canonical metric reported in the methodology brief.
    • Right-rail POOLED AUC ≈ 0.75 at default — single AUC across the union of ~84k unique-tract predictions (one prediction per tract, dominated by F8's 2023 test window plus a smaller tail from earlier folds). It treats all visible tracts as one rank-ordering instead of averaging across folds.

    The pooled number is lower because mixing F1→F8 predictions in one rank shifts the positive/negative ratio relative to any single fold. As you filter (state, race, risk band), the pooled AUC moves further as the positive count and the score variance change.

    If POOLED AUC reads n=37 · too few, that means n_labeled ≤ 50 — Mann-Whitney AUC becomes too noisy to report below that threshold.

    DATA PANEL

    Tract-year panel covering 2009–2024 (16 years). Sources: FFIEC CRA disclosure (full backfill 2009–2024, 5.5 GB); FDIC Summary of Deposits via API (1.43M branch-years); FFIEC HMDA via CFPB Data Browser API (121.7M LAR rows aggregated to 550k tract-state-years, 2018–2024 only); Census ACS 5-year (245k tract-vintages); USDA RUCA + County Typology; Census tract crosswalks (2000↔2010 + 2010↔2020) for vintage harmonization onto the 2020 baseline.

    Round 4 used a 6-year panel with one chronological split → AUC 0.751. Round 5's +0.106 AUC comes from longer panel + walk-forward + closed leakage paths, not from architecture (we ran XGBoost + LightGBM + Optuna; tuning was a wash).
